# JAP Plus 项目模块说明

## 项目概述

JAP Plus 是一个基于 Node.js + TypeScript 开发的智能需求分析系统，集成了大语言模型（LLM）和 Model Context Protocol（MCP）工具，用于自动化需求引出、分析和文档生成。

## 目录结构

```
├── src/
│   ├── constants/          # 常量定义
│   ├── http/               # API 路由和 WebSocket
│   ├── nodes/              # 工作流节点
│   ├── runtime/            # 运行时相关
│   ├── services/           # 服务层
│   ├── state/              # 状态管理
│   ├── tools/              # 工具类
│   ├── index.ts            # 入口文件
│   └── server.ts           # 服务器配置
├── public/                 # 静态文件
├── logs/                   # 日志文件
├── package.json            # 项目配置
└── tsconfig.json           # TypeScript 配置
```

## 模块详细说明

### 1. constants 模块

#### domainConstants.ts

- 定义领域相关的常量
- 包含问题维度（QUESTION_DIMENSIONS）等核心概念
- 为需求分析提供标准化的分类体系

#### promptTexts.ts

- 定义大语言模型的系统提示文本
- 包含 API_ELICITATION_PROMPT（需求引出提示）
- 包含 API_FINALIZE_PROMPT（需求最终化提示）
- 包含 DEEP_THINKING_SYSTEM_PROMPT（深度思考提示）

#### logTexts.ts

- 定义日志相关的常量文本
- 为系统日志提供标准化的消息格式

### 2. http 模块

#### configRoutes.ts

- 处理系统配置相关的 API 路由
- 提供 LLM 测试、工作区选择等功能
- 管理系统级别的配置参数

#### elicitationRoutes.ts

- **核心功能**：处理需求引出和最终化的 API 路由
- **问卷生成**：`/api/v1/elicitation/questionnaire` 端点
  - 接收用户需求，生成结构化问卷
  - 支持快速模式和深度模式
  - 处理 LLM 调用失败的备用方案
- **需求最终化**：`/api/v1/elicitation/finalize` 端点
  - 整合问卷答案，生成最终需求文档
  - 尝试使用 MCP 工具生成 PRD 草稿
  - 提供详细的元数据和诊断信息

#### taskRoutes.ts

- 管理任务工作流的 API 路由
- 处理文件生成、任务状态更新等功能
- 维护任务历史记录

#### websocket.ts

- 实现 WebSocket 服务器
- 用于实时广播工作流事件和日志
- 提供前端与后端的实时通信机制

### 3. nodes 模块

#### elicitationNode.ts

- 定义需求引出工作流节点
- 处理需求分析的核心逻辑
- 与其他工作流节点集成

### 4. runtime 模块

#### skillContext.ts

- 加载和管理技能上下文
- 为需求分析提供领域知识支持
- 整合外部技能信息

#### workflowEvents.ts

- 管理工作流事件的触发和广播
- 与 WebSocket 系统集成
- 提供事件驱动的系统架构

### 5. services 模块

#### elicitationHelpers.ts

- 提供需求引出相关的辅助函数
- 包含问题去重、签名生成等功能
- 处理 PRD 草稿的标准化

#### structuredOutputFallback.ts

- 处理结构化输出的备用方案
- 确保 LLM 输出符合预期格式
- 提供错误处理和恢复机制

### 6. state 模块

#### japState.ts

- 定义系统状态的数据结构
- 使用 Zod 进行数据验证
- 包含问卷、问题等核心数据模型

### 7. tools 模块

#### mcpClient.ts

- 实现 MCP（Model Context Protocol）客户端
- 与 MCP 工具进行交互
- 提供项目上下文读取、PRD 生成等功能

### 8. 核心文件

#### index.ts

- 项目的主入口文件
- 初始化系统组件
- 导出核心功能

#### server.ts

- 配置和启动 Express 服务器
- 注册 API 路由和 WebSocket
- 处理静态文件和中间件

## 核心流程

### 1. 需求引出流程

1. 接收用户需求和上下文信息
2. 加载项目上下文和技能上下文
3. 根据模式（快速/深度）生成问卷
4. 使用 LLM 生成结构化问卷
5. 对生成的问题进行去重处理
6. 返回问卷结果和元数据

### 2. 需求最终化流程

1. 接收原始需求、问卷和答案
2. 尝试使用 MCP 工具生成 PRD 草稿
3. 加载技能上下文并构建诊断信息
4. 将相关数据写入草稿文件
5. 使用 LLM 整合所有信息生成最终需求
6. 返回最终需求和详细元数据

### 3. 实时通信流程

1. 前端建立 WebSocket 连接
2. 后端广播工作流事件和日志
3. 前端实时更新界面状态
4. 提供实时反馈和进度跟踪

## 核心模块详细流程

### 1. 需求引出模块 (elicitationRoutes.ts)

#### 问卷生成流程
1. **接收请求**：接收包含原始需求、上下文信息和LLM配置的POST请求
2. **参数验证**：验证必需字段（需求内容、LLM API密钥）
3. **加载上下文**：
   - 加载项目上下文（通过MCP客户端）
   - 加载技能上下文
4. **深度思考（可选）**：
   - 如果是深度模式，调用顺序思考工具
   - 生成思考轨迹并添加到项目上下文中
5. **生成问卷**：
   - 构建包含需求、上下文、历史问答等信息的payload
   - 使用LLM生成结构化问卷
   - 对生成的问题进行去重处理
6. **返回结果**：
   - 返回问卷结果、清晰度状态和元数据
   - 包含时间消耗、批量信息等统计数据
7. **错误处理**：
   - 当LLM调用失败时，返回备用问卷
   - 提供错误原因和备用标志

#### 需求最终化流程
1. **接收请求**：接收包含需求、问卷、答案和LLM配置的POST请求
2. **参数验证**：验证必需字段（需求内容、LLM API密钥）
3. **生成PRD草稿**：
   - 尝试使用MCP工具生成PRD草稿
   - 对PRD草稿进行标准化处理
4. **构建诊断**：
   - 分析MCP工具的可用性和执行状态
   - 生成详细的诊断信息
5. **写入草稿文件**：
   - 将原始草稿、标准化草稿和输入信息写入文件
   - 保存MCP状态和诊断信息
6. **生成最终需求**：
   - 构建包含所有信息的payload
   - 使用LLM整合信息生成最终需求文档
   - 更新最终需求文件
7. **返回结果**：
   - 返回最终需求和详细的元数据
   - 包含MCP状态、诊断信息和文件路径

### 2. WebSocket模块 (websocket.ts)

#### 实时通信流程
1. **服务器初始化**：
   - 在Express服务器上创建WebSocket服务器
   - 配置WebSocket连接处理
2. **连接管理**：
   - 处理新的WebSocket连接
   - 维护活跃连接列表
   - 处理连接关闭事件
3. **事件广播**：
   - 监听工作流事件
   - 将事件实时广播给所有连接的客户端
   - 支持不同类型的事件（如日志、状态更新等）
4. **错误处理**：
   - 处理WebSocket连接错误
   - 确保系统稳定性

### 3. MCP客户端模块 (mcpClient.ts)

#### 项目上下文读取流程
1. **客户端初始化**：
   - 创建或获取共享MCP客户端实例
   - 配置客户端参数
2. **上下文读取**：
   - 调用MCP工具读取项目上下文
   - 处理读取结果和错误
3. **工具发现**：
   - 列出可用的MCP工具
   - 识别PRD生成相关工具

#### PRD草稿生成流程
1. **参数准备**：
   - 收集产品名称、描述、核心功能等信息
   - 整理约束条件和附加上下文
2. **工具调用**：
   - 选择合适的PRD生成工具
   - 调用工具生成PRD草稿
3. **结果处理**：
   - 处理工具返回的PRD草稿
   - 提取工具名称和生成状态

### 4. 工作流事件模块 (workflowEvents.ts)

#### 事件管理流程
1. **事件定义**：
   - 定义系统中使用的事件类型
   - 建立事件命名规范
2. **事件触发**：
   - 提供事件触发接口
   - 支持不同级别的事件（如信息、警告、错误）
3. **事件广播**：
   - 与WebSocket模块集成
   - 将事件广播给所有连接的客户端
4. **事件日志**：
   - 记录事件到日志系统
   - 提供事件历史查询功能

### 5. 需求引出辅助模块 (elicitationHelpers.ts)

#### 问题处理流程
1. **问题签名生成**：
   - 为每个问题生成唯一签名
   - 用于问题去重和跟踪
2. **问题去重**：
   - 基于签名识别重复问题
   - 保留唯一问题，移除重复项
3. **答案处理**：
   - 将答案转换为标准化格式
   - 处理不同类型的答案（单选、多选等）

#### PRD草稿标准化流程
1. **格式标准化**：
   - 处理PRD草稿的格式
   - 确保草稿符合预期结构
2. **内容清理**：
   - 移除不必要的内容
   - 标准化标题、列表等格式
3. **质量检查**：
   - 检查草稿的完整性
   - 确保关键信息不缺失

### 6. 服务器模块 (server.ts)

#### 服务器启动流程
1. **应用初始化**：
   - 创建Express应用实例
   - 配置CORS、JSON解析和静态文件服务
2. **路由注册**：
   - 注册配置路由
   - 注册需求引出路由
   - 注册任务管理路由
3. **WebSocket配置**：
   - 创建HTTP服务器
   - 注册WebSocket服务
4. **服务器启动**：
   - 监听指定端口
   - 输出启动信息

## 技术栈

- **后端**：Node.js + TypeScript + Express
- **LLM 集成**：LangChain + OpenAI
- **实时通信**：WebSocket
- **数据验证**：Zod
- **MCP 集成**：@modelcontextprotocol/sdk
- **开发工具**：tsx, TypeScript

## 关键特性

1. **智能化需求分析**：利用 LLM 生成结构化问卷和最终需求
2. **多模式分析**：支持快速分析和深度分析两种模式
3. **MCP 工具集成**：利用外部工具增强需求分析能力
4. **实时反馈**：通过 WebSocket 提供实时事件和日志
5. **错误处理**：实现完善的错误处理和备用方案
6. **数据持久化**：将中间过程和结果写入文件，便于追溯
7. **性能监控**：记录各个环节的时间消耗，便于优化

## 总结

JAP Plus 是一个功能完整的智能需求分析系统，通过集成 LLM 和 MCP 工具，实现了需求引出、分析和文档生成的自动化流程。系统采用模块化设计，各组件职责清晰，通过实时通信机制为用户提供良好的交互体验。

该项目展示了如何将大语言模型与传统软件工程实践相结合，为需求分析和文档生成提供智能化解决方案。
